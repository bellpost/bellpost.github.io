导语：如何使用vmstat和iostat工具帮助我们了解操作系统状态。

vmstat和iostat命令工具可以帮助我们了解操作系统状态。iostat和vmstat显示的第一行一般是系统从开机到现在的统计信息，一般不具有参考意义，请注意。

vmstat用法

vmstat这个命令展示的信息非常全面，涵盖系统的各个方面。

vmstat 3 3 第一个3设置每3秒刷新一次，后面一个3设置刷新次数。

注意内存、交换区、I/O统计是块数而不是字节。块大小通常是1024字节。

参数含义

  • procs

    • r列显示多少进程在等待CPU
    • b列显示多少进程在阻塞状态（等待I/O）

  • memory

    • swpd 虚拟内存的使用量，如果不为0，则物理内存不足；（页面交换）
    • free 空闲内存
    • buff 用作缓冲区的内存
    • cache 缓存内存使用量

  • swap 显示每秒有多少块被换入磁盘，多少块换出磁盘

    • si 换入磁盘
    • so 换出磁盘

  • io 反映磁盘I/O

    • bi 从块设备中读取
    • bo 从块设备中写出

  • system

    • in 每秒中断的数量
    • cs 每秒上下文切换的数量

  • cpu 显示CPU花费在各类操作的百分比

    • us 执行用户代码花费的时间（非内核）
    • sy 执行系统代码（内核）
    • id 空闲时间
    • wa 等待I/O时间
    • st 虚拟机偷走的时间（虚拟化）

iostat用法

iostat -c 可以查看到CPU信息，但是我们主要还是使用这个命令的-d选项，查看设备统计信息。后面的5也是每5秒刷新一次。

参数含义

  • rrqm/s 每秒合并到设备中的读取请求的数量（合并是将多个逻辑请求合并为一个请求到实际磁盘）
  • wrqm/s 每秒合并到设备中的写入请求的数量
  • r/s 每秒发送到设备的读请求
  • w/s 每秒发送到设备的写请求
  • rkB/s 每秒从设备读取的扇区数
  • wkB/s 每秒从设备写入的扇区数
  • avgrq-sz 发出到设备的请求的平均大小（以扇区为单位）。
  • avgqu-sz 发出到设备的请求的平均队列长度。
  • await 磁盘排队花费的毫秒数
  • r_await 磁盘读请求排队花费的毫秒数
  • w_await 磁盘写请求排队花费的毫秒数
  • svctm 服务请求花费的时间，不包括排队时间
  • %util 向设备发出I/O请求的经过时间百分比，串行时接近100%时设备饱和，并行时不能反映设备极限。

计算请求服务并发数

读写的单位是秒，服务时间单位是千分之一毫秒。

计算方式一：
concurrency = （r/s + w/s）* (svctm / 1000)

利特尔法则（英语：Little's law），基于等候理论，由约翰·利特尔在1954年提出。利特尔法则可用于一个稳定的、非占先式的系统中。其内容为：
在一个稳定的系统（L）中，长期的平均顾客人数，等于长期的有效抵达率（λ），乘以顾客在这个系统中平均的等待时间（W）；----维基百科

计算方式二：
concurrency = (avuqu_sz * svctm) / await

通过平均队列大小、服务时间、以及平均等待时间。
